Windows:
Baixe o instalador do Git para Windows no site oficial: Git for Windows.

Execute o arquivo baixado e siga as instruções de instalação padrão. Certifique-se de selecionar todas as opções padrão, a menos que você saiba o que está fazendo.

Linux (Debian/Ubuntu):
Abra o terminal.

Execute o seguinte comando para instalar o Git:
sudo apt-get update
sudo apt-get install git

Configurando o Git:
Antes de começar a usar o Git, é importante configurar seu nome de usuário e endereço de email. Isso será usado para identificar seus commits. Execute os seguintes comandos no terminal, substituindo "Seu Nome" e "seu@email.com" pelos seus próprios dados:
git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

Criando um Repositório:
Para começar a usar o Git, você precisa de um repositório. Você pode criar um novo repositório ou clonar um existente.

*Criando um Repositório Local:*
Crie uma pasta para o seu projeto:
mkdir meu-projeto
cd meu-projeto

Inicialize um repositório Git na pasta:
git init

Clonando um Repositório Existente:
Se você deseja contribuir para um projeto existente ou apenas trabalhar com um repositório remoto, pode cloná-lo:
git clone URL_DO_REPOSITÓRIO

Fazendo um Commit:
Agora que você tem um repositório Git, pode começar a trabalhar nos arquivos e fazer commits para rastrear as alterações.

Adicione arquivos ao stage (área de preparação) usando o comando git add:
git add nome-do-arquivo

ou para adicionar todos os arquivos modificados:
git add .

Faça um commit com uma mensagem descritiva:
git commit -m "Mensagem do commit"

*Aqui estão alguns comandos Git básicos que você pode usar:*

Verifique o status dos arquivos no seu repositório:
git status

Visualize o histórico de commits:
git log

Crie uma nova branch (ramificação):
git branch nome-da-branch

Mude para uma branch específica:
git checkout nome-da-branch

Faça o merge de uma branch na branch atual:
git merge nome-da-branch

Atualize seu repositório local com as alterações do repositório remoto:
git pull origin nome-da-branch

Envie suas alterações para o repositório remoto:
git push origin nome-da-branch
